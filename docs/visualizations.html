What the three Summary tabs actually show

Every number below came from a real run of python render/test.py against the engine's built-in mock (300 synthetic workers, Mar–Jun 2026) — nothing here is hand-typed. Locked definitions live in render/test.py's module docstring; this page is a worked example of what those definitions produce, shaped exactly as it lands in each central-sheet tab.

Summary_1 — show-up buckets

Written by showup_block(), all 4 departments combined ("All SOCN" scope).

A show-up day = one clock-in on one date. A worker's day-count for a month is the number of distinct dates they appeared, any team, combined.

Every worker that month falls into exactly one bucket by that count: 1-5, 6-10, 11-15, 16-20, 21-30. The % columns are each bucket's share of that month's total bucketed workers ("Sum Month") — not a fixed denominator across months.

The engine can also scope this to one station (showup_block(df, "CBS")) — e.g. CBS in March skews much more junior: 74.42% of CBS workers land in the 1-5 bucket that month, vs. 50.55% department-wide. run_sync() only calls the ALL-SOCN form, so per-station breakdowns exist in the engine but never reach the central sheet today.

Summary_Rotation — station rotation

Written by rotation_summary(): one row per (month, team), all 4 months.

Per worker per month: days worked at each team. Rotation count = how many different teams they touched that month. A worker is Rotated for team T only if both hold: count[T] > 1 and Rotation count > 1 — more than one day at T, and more than one team overall. One stray day elsewhere doesn't count as rotating.

oneday_nonrot ("Reading B") = of the non-rotated group, workers who touched that team exactly one day — barely there, not a real team member. rotation%/non_rotation% are out of population; oneday% is out of non_rotation specifically.

Summary_5 — consecutive-day streaks

Written by streak_month_crosstab(): one row per month.

Consecutive = calendar-adjacent dates; a streak can't cross a month boundary. Per worker per month: days = total distinct show-up days, run = their longest consecutive streak that month.

Bucketed by days: <10 / >10. Separately flagged used to if run >= 3 that month, else never. The tab crosstabs those two axes; active = total workers counted that month.

The engine also has a weekly view (streak_week() — same ≥3-day-run test, bounded to one ISO week instead of a month) and a "depth" view (weeks_depth() — of workers who ever hit that streak, how many separate weeks they did it in). Neither is called from run_sync(), so only the monthly crosstab above reaches the actual Summary_5 tab today.
